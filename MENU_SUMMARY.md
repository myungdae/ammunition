## 🔗 온톨로지 시스템
- **25개 노드**: Depot, AmmoType, InventoryLine, Batch, Events, Signals
- **40+ 관계**: hasLocation, tracksBatch, producedBy 등
- 인터랙티브 지식 그래프
- 노드 상세 정보 및 연결 관계 탐색
- 컬러 코딩 (Core, Event, Policy, Signal, Audit)

## 📐 시스템 아키텍처
- **Layer 1**: Data/Sensor (데이터 수집 - IoT, RFID, API)
- **Layer 2**: Ontology Core (25개 클래스, 40+ 관계)
- **Layer 3**: SWRL Reasoning (AI 추론 - 패턴 매칭, 상황 분석)
- **Layer 4**: SHACL Control (근거 검증 - 데이터 완전성)
- **Layer 5**: Governance Ontology (제안/승인/감사 구조화)
- **Layer 6**: Human Decision (승인/보류/거부)
- Palantir 3단계 vs. 본 시스템 6단계 비교

## 🎯 핵심 흐름

1. **데이터 수집** (Layer 1) → 센서/RFID/트랜잭션
2. **온톨로지 매핑** (Layer 2) → 지식 그래프 구축
3. **AI 분석** (Layer 3) → 상황 분석 및 제안 생성
4. **검증** (Layer 4) → 근거 충족도 확인
5. **제안** (Layer 5) → ActionProposal 생성
6. **승인** (Layer 6) → 지휘관 최종 결정
7. **실행** → 자동화된 액션 수행
8. **모니터링** → 대시보드 및 알림
